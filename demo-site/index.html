What makes it different

Hierarchical, not flat

Most tools output 4–6 stems. StemForge AI goes deeper by separating each group into its sub-components in a second pass.

🎤

Vocal Split

Lead vocal vs. backing vocals/harmonies — separated using Wiener soft masking + mid-side processing.

🥁

Drum Detail

Kick, snare, hi-hat, cymbals, toms extracted individually using HPSS + frequency band masking.

🎸

Full 6-Stem Base

Demucs htdemucs_6s: vocals, drums, bass, guitar, piano/synth, other.

🎛️

Web Mixer

Play all stems in sync. Solo, mute, adjust volume, seek — directly in the browser.

⚡

Async Pipeline

FastAPI + Celery + Redis. Upload once, worker processes in background, results stream as they're ready.

📦

Docker Ready

Single docker compose up starts everything: API, worker, Redis, frontend.

Limitations & Future Work

Honest notes

  • Live AI processing requires significant CPU (5–15 min/song) or a GPU. Not suitable for shared hosting.
  • Backing vocal separation is experimental — mid-side masking works well on professionally mixed tracks.
  • Drum separation uses spectral methods, not a dedicated ML model (LarsNet integration is planned).
  • Future: GPU support (DEVICE=cuda), LarsNet drums, dedicated vocal harmony model.
  • Future: User accounts, persistent storage, public API.
